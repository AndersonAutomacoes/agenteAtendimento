  Criar reconhecimento msg de audio
    - A aplicação deve transcrever o audio para mensagem e enviar para o agente para que ele possa interagir com o cliente    
	-Sugestão de Melhoria: * Custo e Performance: Use o modelo Whisper (OpenAI) via API ou uma biblioteca local se o servidor permitir.
	-Feedback ao Usuário: Como a transcrição leva alguns segundos, instrua o Agente a enviar uma reação ou uma pequena mensagem ("Processando seu áudio...") para evitar que o usuário ache que o bot travou.

Validação: Se a transcrição for muito curta ou inconclusiva (ruído de oficina), o bot deve pedir gentilmente para repetir ou digitar.
  Agendar apenas Serviços oferecidos pelo cliente
   - Deverá existir na aplicação web uma aba para cadastro e gerenciamento de Serviços que podem ser agendados pelo Cliente, esses serviços ficarão em uma tabela tenant_services

   - Para o agendamento ser confirmado deverá antes um desses serviços devem serem escolhidos
   - O agente deverá informar na mensagem de confirmação de Agendamento ou de Cancelamento desse agendamento os serviços disponibilizados pelo cliente 
   - Sugestão de Melhoria: * ID vs Nome: No banco tenant_appointments, armazene o service_id (FK para tenant_services) e não apenas o nome string. Isso facilita relatórios futuros (ex: "Qual serviço é mais lucrativo?").
